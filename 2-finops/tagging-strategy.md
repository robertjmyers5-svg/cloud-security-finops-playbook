# Tagging Strategy

Tagging is required for allocation, visibility, automation, and governance.

---

# Required Tags
- owner  
- application  
- environment  
- cost-center  
- compliance  

# Optional Tags
- project  
- service-owner  
- region  
- sensitivity-level  
- retention-policy  

---

# Tag Format Standard
Use lowercase, consistent keys, no spaces.

Examples:
- owner=john.doe  
- cost-center=finance  
- environment=prod  

---

# Tag Lifecycle
1. Define  
2. Approve  
3. Apply  
4. Validate  
5. Review  
6. Retire
