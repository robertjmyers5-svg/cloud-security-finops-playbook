Incident Response Labs

Lab 1: Compromised IAM User Simulation
- Create test IAM user
- Simulate suspicious API calls
- Revoke session and rotate keys
- Investigate logs

Lab 2: Azure Identity Compromise
- Trigger risky sign-in
- Use Entra ID protection and Conditional Access
- Lock down user roles

Lab 3: Quarantine a Cloud VM
- Create quarantine security group or NSG
- Move compromised VM to isolated network
- Capture disk snapshots

Lab 4: Detect and Contain Public Exposure
- Identify publicly exposed S3 bucket or storage account
- Block access
- Implement policies to prevent recurrence

Lab 5: CloudTrail Investigation
- Search for privilege escalation events
- Export logs for forensic review
