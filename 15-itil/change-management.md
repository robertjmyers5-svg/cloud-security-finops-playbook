Change Management

Purpose:
Ensure cloud changes are controlled, approved, documented, and safe.

Types of Changes:
- Standard changes (pre-approved)
- Normal changes (reviewed)
- Emergency changes (fast-tracked)

Cloud Mapping:
- Maps to IaC deployments (Terraform, Bicep)
- CI/CD approvals
- Azure DevOps or GitHub Pull Requests
- Production firewall rule updates
- IAM permission changes

Key Principles:
- Minimize outages
- Reduce misconfigurations
- Track all changes
- Improve auditability
